# Sub EQ — 超低频参量均衡器 架构设计方案

## 1. 项目概述

### 1.1 目标
构建一个覆盖 **0.5Hz ~ 500Hz** 频段的标准图形化参量 EQ VST3 插件，功能与行为和普通音频参量 EQ 完全一致，仅工作频段偏移至次声/超低频范围。

### 1.2 输出规格
- **格式**: VST3（当前 Projucer 已配置）
- **输入**: 完整音频信号
- **输出**: 经 0.5~500Hz 频段塑形后的完整音频信号
- **通道**: 仅立体声（2进2出）

---

## 2. 核心挑战与对策

### 2.1 系数精度（关键）

在 48kHz 采样率下，0.5Hz 处的归一化频率约为 `0.5 / 48000 ≈ 0.0000104`。

| 问题 | 影响 | 对策 |
|------|------|------|
| 单精度 `float` 计算双线性变换系数 | 频率偏移可达数 Hz，Q 值失真 | **全程使用 `double` 计算系数** |
| 单精度滤波器状态变量累积误差 | DC 漂移、低频失真 | **滤波器内部运算使用 `double`** |
| 浮点去规范化（denormals） | CPU 占用 spike | `ScopedNoDenormals` + 已启用 |

**处理链路**: `AudioBuffer<float>` → 转 `double` → 双精度 Biquad 级联 → 转回 `float` → 输出

### 2.2 频率参数映射

线性旋钮在 0.5~500Hz 范围几乎无法使用（90% 行程集中在 450~500Hz）。

**采用对数映射**：
```cpp
NormalizedRange(0.5f, 500.0f, 0.0f, 0.5f) // skew=0.5 实现近似对数刻度
```

## 3. DSP 架构设计

### 3.1 滤波器类型支持

支持 8 种标准参量 EQ 节点类型：

| 类型 | 英文标识 | Biquad 数量 | 用途 |
|------|----------|-------------|------|
| 峰值/钟形 | Bell | 1 | 中心频率提升/衰减 |
| 低切/高通 | HighPass | 1~2 | 滤除超低频成分 |
| 高切/低通 | LowPass | 1~2 | 限制高频通过 |
| 低频搁架 | LowShelf | 1 | 低频整体提升/衰减 |
| 高频搁架 | HighShelf | 1 | 高频整体提升/衰减 |
| 陷波 | Notch | 1 | 特定频率点完全切除 |
| 倾斜 | Tilt | 2（级联） | 整体频响倾斜 |
| 带通 | BandPass | 1 | 只保留特定频段 |

> **Tilt 实现**: LowShelf(A) + HighShelf(1/A) 两个 Biquad 级联，等效于整体频响绕中心频率倾斜。

### 3.2 双精度 Biquad 实现

采用 **转置直接 II 型（Transposed Direct Form II）** 结构：

```cpp
struct BiquadState {
    double z1 = 0.0, z2 = 0.0;

    double process(double in, const BiquadCoefficients& c) {
        double out = c.b0 * in + z1;
        z1 = c.b1 * in - c.a1 * out + z2;
        z2 = c.b2 * in - c.a2 * out;
        return out;
    }
};
```

> **变更原因**：标准 Direct Form II 在 0.5Hz + 低 Q 的极端系数下，中间变量 `w` 会指数增长导致数值不稳定。Transposed DF2 动态范围更小，且配合 per-channel 独立状态后完全消除了通道间串扰。

### 3.3 节点级联架构

```
输入样本 (float)
    ↓ 转为 double
[Biquad Node 1] ──→ [Biquad Node 2] ──→ ... ──→ [Biquad Node N]
    ↓ 转为 float
输出样本
```

所有节点串联处理，每个节点根据类型配置 1 或 2 个 Biquad。

### 3.4 系数更新策略

- **快照更新**: 参数改变时立即计算新系数并替换。未实现跨样本平滑插值（增益变化可能产生轻微 click，建议在宿主中使用自动化平滑）
- **线程安全**: 系数更新仅在音频线程（`processBlock` 中调用 `updateEQParameters()`）进行。GUI 线程通过 APVTS 修改参数值，但系数计算延迟到下一次音频回调，避免多线程竞争修改 Biquad 系数
- **浮点抖动过滤**: 使用 epsilon 容差（0.001）比较参数值，避免 APVTS 原子值的微小抖动触发无意义的系数重算

## 4. 参数系统设计

### 4.1 全局参数

| 参数 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| Master Gain | Float | -24.0 ~ +24.0 dB | 0.0 | 总输出增益（右侧边缘垂直Slider） |
| Bypass | Bool | true/false | false | 硬旁路 |

### 4.2 节点参数（每节点，共 8 个）

| 参数 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| Node N Enabled | Bool | true/false | **false** | 节点开关（初始未启用） |
| Node N Frequency | Float | 0.5 ~ 500.0 Hz | 100.0 | 中心/截止频率（对数刻度） |
| Node N Gain | Float | -24.0 ~ +24.0 dB | 0.0 | 增益 |
| Node N Q | Float | 0.1 ~ 10.0 | 0.707 | 品质因数 |
| Node N Type | Choice | 8 types | Bell | 滤波器类型 |

> **参数 ID 命名**: `nodeN_freq`, `nodeN_gain`, `nodeN_q`, `nodeN_type`, `nodeN_enabled`

### 4.3 参数布局（APVTS）

使用 `AudioProcessorValueTreeState` 统一管理，便于：
- 宿主自动化
- 状态保存/恢复
- GUI 与 DSP 的线程安全通信

## 5. 图形界面设计

### 5.1 布局方案：FabFilter Pro-Q2 风格

窗口尺寸：**900 × 560 像素**

```
┌────────────────────────────────────────────────────┬──────────┐
│  [频响曲线区域 — 对数频率轴 0.5~500Hz, 840×500]    │          │
│  · 深灰背景 (#2a2a2a)                              │ [Master  │
│  · 品红色频响曲线 (#e040fb) + 青色相位曲线         │  Gain    │
│  · 实时频谱填充（半透明品红色）                     │  Slider] │
│  · 白色节点：实心圆（选中）/ 空心圆环（未选中）     │   60px   │
│  · 选中节点展开参数标签                              │          │
│  · 网格线 + 频率/增益/相位刻度                      │          │
└────────────────────────────────────────────────────┴──────────┘
```

### 5.2 频响曲线绘制

- **频率轴**: 对数刻度，范围 0.5 ~ 500Hz，关键频率标注（1, 10, 100, 500Hz）
- **增益轴**: 固定范围 ±24dB，0dB 参考线高亮
- **曲线颜色**: 单一颜色（如橙色/青色），选中节点对应曲线段高亮
- **曲线计算**: 对每个像素列，累加所有启用节点的复数频响，转换为 dB
- **性能**: 仅在参数变化时重算曲线，使用 `juce::Path` 缓存

### 5.3 节点交互（Pro-Q2 风格）

| 操作 | 行为 |
|------|------|
| 左键单击空白处 | 在光标位置新建节点（默认 Bell），选中该节点 |
| 左键单击 + 按住拖动 | 新建节点后直接进入拖动模式 |
| 左键单击已有节点 | 选中该节点（取消上一个选中） |
| 左键拖拽节点 | 同时调节 Frequency 和 Gain（Bell/Shelf/Tilt）或 Frequency 和 Q（HP/LP/Notch/BP） |
| 滚轮在节点上 | 调节 Q 值 |
| 右键单击节点 | 删除该节点 |
| 右键按住 + 拖动 | 删除拖动路径上经过的所有节点 |
| Shift+拖拽 | 仅调节 Frequency（锁定 Gain/Q） |
| Ctrl+拖拽 | 仅调节 Gain（Bell/Shelf/Tilt）或仅调节 Q（HP/LP/Notch/BP） |
| 双击节点 | 重置 Gain 为 0dB + Q 为 0.707（Bell/Shelf/Tilt）；仅重置 Q 为 0.707（HP/LP/Notch/BP） |

### 5.4 节点参数标签（选中时展开）

选中节点时在节点附近展开浮动标签，显示：
- **频率**: 显示如 "100.0 Hz"，单击进入文本编辑
- **增益**: 显示如 "+6.0 dB"，单击进入文本编辑
- **Q 值**: 显示如 "Q: 1.00"，可由滚轮调节，也可单击文本编辑
- **类型**: 显示当前类型名称（如 "Bell"），单击弹出下拉菜单选择类型

标签自动避让边界，避免超出频响区域。节点失选时标签收起。

### 5.5 总体增益控制

位于界面右侧边缘的垂直 Slider：
- 范围: -24dB ~ +24dB
- 默认: 0dB
- 显示当前数值标签
- 不参与频响曲线显示（纯增益，不影响曲线形状）

## 8. 关键设计决策（已确认）

| 决策项 | 确认结果 |
|--------|----------|
| EQ 节点数量 | **8 个**，初始状态无节点，曲线平直 |
| 增益范围 | **±24dB**（节点 + 总增益一致） |
| GUI 风格 | **FabFilter Pro-Q2 风格**，左键新建/拖动，右键删除，滚轮调Q，单击选中 |
| 通道支持 | **仅立体声**（2进2出） |
| Interface 类 | **移除**，完全手写 GUI |
| 窗口尺寸 | **900 × 560 像素**，右侧 60px 总增益 Slider |
| 颜色主题 | **深灰背景 (#2a2a2a) + 品红色频响曲线 (#e040fb) + 青色相位曲线 + 白色节点 (#ffffff)** |
| 节点外观 | **统一外径**：选中为白色实心圆，未选中为白色空心圆环（不随 Q 变化） |
| 节点默认状态 | **全部禁用**（false），初始无活动节点 |
| 类型切换 | **弹出下拉菜单**（8 种类型列表选择） |
| 键盘操作 | **不需要** |
| 实时频谱 | **1/6 倍频程**，Catmull-Rom 样条曲线绘制 |
| ASIO 支持 | **已添加**，Standalone 版本可用 |
