# Sub EQ — 超低频参量均衡器 架构设计方案

## 1. 项目概述

### 1.1 目标
构建一个覆盖 **0.5Hz ~ 500Hz** 频段的标准图形化参量 EQ VST3 插件，功能与行为和普通音频参量 EQ 完全一致，仅工作频段偏移至次声/超低频范围。

### 1.2 输出规格
- **格式**: VST3（当前 Projucer 已配置）
- **输入**: 完整音频信号
- **输出**: 经 0.5~500Hz 频段塑形后的完整音频信号
- **通道**: 仅立体声（2进2出）

---

## 2. 核心挑战与对策

### 2.1 系数精度（关键）

在 48kHz 采样率下，0.5Hz 处的归一化频率约为 `0.5 / 48000 ≈ 0.0000104`。

| 问题 | 影响 | 对策 |
|------|------|------|
| 单精度 `float` 计算双线性变换系数 | 频率偏移可达数 Hz，Q 值失真 | **全程使用 `double` 计算系数** |
| 单精度滤波器状态变量累积误差 | DC 漂移、低频失真 | **滤波器内部运算使用 `double`** |
| 浮点去规范化（denormals） | CPU 占用 spike | `ScopedNoDenormals` + 已启用 |

**处理链路**: `AudioBuffer<float>` → 转 `double` → 双精度 Biquad 级联 → 转回 `float` → 输出

### 2.2 频率参数映射

线性旋钮在 0.5~500Hz 范围几乎无法使用（90% 行程集中在 450~500Hz）。

**采用对数映射**：
```cpp
NormalizedRange(0.5f, 500.0f, 0.0f, 0.5f) // skew=0.5 实现近似对数刻度
```

## 3. DSP 架构设计

### 3.1 滤波器类型支持

支持 7 种标准参量 EQ 节点类型：

| 类型 | 英文标识 | Biquad 数量 | 用途 |
|------|----------|-------------|------|
| 峰值/钟形 | Bell | 1 | 中心频率提升/衰减 |
| 低切/高通 | HighPass | 1~2 | 滤除超低频成分 |
| 高切/低通 | LowPass | 1~2 | 限制高频通过 |
| 低频搁架 | LowShelf | 1 | 低频整体提升/衰减 |
| 高频搁架 | HighShelf | 1 | 高频整体提升/衰减 |
| 陷波 | Notch | 1 | 特定频率点完全切除 |
| 倾斜 | Tilt | 2（级联） | 整体频响倾斜 |

> **Tilt 实现**: LowShelf(A) + HighShelf(1/A) 两个 Biquad 级联，等效于整体频响绕中心频率倾斜。

### 3.2 双精度 Biquad 实现

采用 **直接 II 型（Direct Form II）** 结构，减少状态变量数量：

```cpp
struct Biquad {
    double b0, b1, b2, a1, a2;  // 系数（double 计算）
    double z1 = 0.0, z2 = 0.0;   // 状态变量

    double process(double in) {
        double w = in - a1 * z1 - a2 * z2;
        double out = b0 * w + b1 * z1 + b2 * z2;
        z2 = z1;
        z1 = w;
        return out;
    }
};
```

### 3.3 节点级联架构

```
输入样本 (float)
    ↓ 转为 double
[Biquad Node 1] ──→ [Biquad Node 2] ──→ ... ──→ [Biquad Node N]
    ↓ 转为 float
输出样本
```

所有节点串联处理，每个节点根据类型配置 1 或 2 个 Biquad。

### 3.4 系数更新策略

- **平滑更新**: 参数改变时，在音频线程内逐样本线性插值过渡到新系数，避免"咔嗒"声
- **快照更新**: 非实时参数（如 Bypass）可立即切换
- **线程安全**: 使用 `std::atomic` 或 `AudioProcessorValueTreeState` 的线程安全机制

## 4. 参数系统设计

### 4.1 全局参数

| 参数 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| Master Gain | Float | -24.0 ~ +24.0 dB | 0.0 | 总输出增益（右侧边缘垂直Slider） |
| Bypass | Bool | true/false | false | 硬旁路 |

### 4.2 节点参数（每节点，共 8 个）

| 参数 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| Node N Enabled | Bool | true/false | true | 节点开关 |
| Node N Frequency | Float | 0.5 ~ 500.0 Hz | 100.0 | 中心/截止频率（对数刻度） |
| Node N Gain | Float | -24.0 ~ +24.0 dB | 0.0 | 增益 |
| Node N Q | Float | 0.1 ~ 10.0 | 0.707 | 品质因数 |
| Node N Type | Choice | 7 types | Bell | 滤波器类型 |

> **参数 ID 命名**: `nodeN_freq`, `nodeN_gain`, `nodeN_q`, `nodeN_type`, `nodeN_enabled`

### 4.3 参数布局（APVTS）

使用 `AudioProcessorValueTreeState` 统一管理，便于：
- 宿主自动化
- 状态保存/恢复
- GUI 与 DSP 的线程安全通信

## 5. 图形界面设计

### 5.1 布局方案：FabFilter Pro-Q2 风格

窗口尺寸：**900 × 500 像素**

```
┌────────────────────────────────────────────────────┬──────────┐
│  [频响曲线区域 — 对数频率轴 0.5~500Hz, 840×500]    │          │
│  · 深灰背景 (#2a2a2a)                              │ [Master  │
│  · 品红色频响曲线 (#e040fb)                        │  Gain    │
│  · 白色节点手柄 (#ffffff)                           │  Slider] │
│  · 选中节点展开参数标签                              │   60px   │
│  · 网格线 + 频率刻度                                │          │
└────────────────────────────────────────────────────┴──────────┘
```

### 5.2 频响曲线绘制

- **频率轴**: 对数刻度，范围 0.5 ~ 500Hz，关键频率标注（1, 10, 100, 500Hz）
- **增益轴**: 固定范围 ±24dB，0dB 参考线高亮
- **曲线颜色**: 单一颜色（如橙色/青色），选中节点对应曲线段高亮
- **曲线计算**: 对每个像素列，累加所有启用节点的复数频响，转换为 dB
- **性能**: 仅在参数变化时重算曲线，使用 `juce::Path` 缓存

### 5.3 节点交互（Pro-Q2 风格）

| 操作 | 行为 |
|------|------|
| 左键单击空白处 | 在光标位置新建节点（默认 Bell），选中该节点 |
| 左键单击 + 按住拖动 | 新建节点后直接进入拖动模式 |
| 左键单击已有节点 | 选中该节点（取消上一个选中） |
| 左键拖拽节点 | 同时调节 Frequency 和 Gain |
| 滚轮在节点上 | 调节 Q 值（节点大小/颜色随 Q 变化） |
| 右键单击节点 | 删除该节点 |
| 右键按住 + 拖动 | 删除拖动路径上经过的所有节点 |
| Shift+拖拽 | 仅调节 Frequency（锁定 Gain） |
| Ctrl+拖拽 | 仅调节 Gain（锁定 Frequency） |
| 双击节点 | 重置 Gain 为 0dB，重置 Q 为 0.707 |

### 5.4 节点参数标签（选中时展开）

选中节点时在节点附近展开浮动标签，显示：
- **频率**: 显示如 "100.0 Hz"，单击进入文本编辑
- **增益**: 显示如 "+6.0 dB"，单击进入文本编辑
- **Q 值**: 显示如 "Q: 1.00"，可由滚轮调节，也可单击文本编辑
- **类型**: 显示当前类型缩写（如 "Bell"），单击循环切换类型

标签自动避让边界，避免超出频响区域。节点失选时标签收起。

### 5.5 总体增益控制

位于界面右侧边缘的垂直 Slider：
- 范围: -24dB ~ +24dB
- 默认: 0dB
- 显示当前数值标签
- 不参与频响曲线显示（纯增益，不影响曲线形状）

## 8. 关键设计决策（已确认）

| 决策项 | 确认结果 |
|--------|----------|
| EQ 节点数量 | **8 个**，初始状态无节点，曲线平直 |
| 增益范围 | **±24dB**（节点 + 总增益一致） |
| GUI 风格 | **FabFilter Pro-Q2 风格**，左键新建/拖动，右键删除，滚轮调Q，单击选中 |
| 通道支持 | **仅立体声**（2进2出） |
| Interface 类 | **移除**，完全手写 GUI |
| 窗口尺寸 | **900 × 500 像素**，右侧 60px 总增益 Slider |
| 颜色主题 | **深灰背景 (#2a2a2a) + 品红色频响曲线 (#e040fb) + 白色节点 (#ffffff)** |
| 类型切换 | **弹出下拉菜单**（7 种类型列表选择） |
| 键盘操作 | **不需要** |
